## T10：Guardrail

**目标：** 实现治理护栏，在 ToolCall 执行前拦截危险命令。

**涉及文件：**
- `src/harness/guardrail.py`（新建）
- `src/tests/test_guardrail.py`（新建）

**实现要点：**
1. `Guardrail` 类，构造接收 `GuardrailConfig`
2. `check_tool_call(tool_call: ToolCall) → GuardrailResult`
3. 只拦截 `RunShell` 类型的 ToolCall（`tool_call.name == "run_shell"`），其他工具放行
4. 从 `tool_call.arguments["command"]` 提取命令字符串，匹配危险模式
5. **内置危险模式：**
   - `rm -rf /`、`rm -rf /*` — 递归删除根目录
   - `dd if=` — 直接磁盘写入
   - `DROP TABLE`、`DROP DATABASE` — 破坏性 SQL
   - `:(){ :|:& };:` — fork 炸弹
   - `> /dev/sda` — 写入块设备
   - `curl ... | bash`、`wget ... -O- | sh` — 网络下载并执行
   - 写入 `~/.ssh/` 或 `~/.config/` — 敏感目录
6. 同时检查 `config.block_list` 中的自定义模式
7. 安全命令（如 `pytest`、`echo`、`python` 等）直接放行

**测试（TDD 红→绿）：**

## T13：AutonomousStopDecision

**目标：** 实现四种停机条件的自主判断，独立为 `AutonomousStopDecision` 类。

**涉及文件：**
- `src/harness/stop_condition.py`（新建）
- `src/tests/test_stop_condition.py`（新建）

**实现要点：**
1. `AutonomousStopDecision` 类，构造接收 `Guardrail` 和 `LoopConfig`
2. `should_stop(context: Context) → StopDecision`：判断四种停机条件
3. `on_finish(tool_call: ToolCall) → StopDecision`：处理 Finish 虚拟工具
4. 四种停机条件：
   - 测试全部通过 → `StopDecision(should_stop=True, success=True, reason="All tests passed")`
   - 达到最大迭代次数 → `StopDecision(should_stop=True, success=False, reason="Max iterations reached")`
   - LLM 返回 Finish → 调用 `on_finish()`，根据 `arguments.reason` 判断成功/失败
   - Guardrail 拦截致命动作 → `StopDecision(should_stop=True, success=False, reason=...)`

**`StopDecision` 模型已在 `models.py` 中定义：**
```python

    reason: str = ""
```
