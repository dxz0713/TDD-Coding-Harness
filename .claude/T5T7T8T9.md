**目标：** 实现兼容 OpenAI Chat Completions API 的 Provider，可对接 OpenAI、DeepSeek、Qwen 等。

**涉及文件：**
- `src/providers/base.py`（修改 — 增加 tools/config 参数）
- `src/providers/mock.py`（修改 — 同步更新 generate 签名）
- `src/providers/openai_compat.py`（新建）
- `src/harness/__init__.py` 或 `src/harness/cli.py`（修改 — 注册 Provider 到 Factory）
- `src/tests/test_providers.py`（追加 — 新测试）

**实现要点：**
1. 继承 `LLMProvider`
2. 构造接收 `api_key: str | None = None`（默认从 `OPENAI_API_KEY` 环境变量读取）
3. `generate(messages, tools=None, config=None)` 调用 OpenAI Chat Completions API
4. `base_url` 从 `config.base_url` 读取（兼容第三方 API）
5. ToolDef 列表转换为 OpenAI tool calling 格式
6. 错误处理：超时 → `LLMTimeoutError`，认证失败 → `LLMAuthError`，速率限制 → `LLMRateLimitError`
7. 确保 Factory 注册：在 `src/harness/__init__.py` 或 CLI 入口中调用 `ProviderFactory.register("openai", OpenAICompatibleProvider)`
