## T11：Main Loop 框架（SP2）

**T11 和 T16 互不依赖，可以并行实现。**

**目标：** 实现主循环骨架，只负责流程控制，所有具体逻辑委托给对应模块。

**涉及文件：**
- `src/harness/loop.py`（新建）
- `src/harness/__init__.py`（修改 — 注册工具到 Dispatcher）
- `src/tests/test_loop.py`（新建）

**设计原则：**
1. **依赖注入**：Loop 不创建任何依赖，全部通过构造注入
2. **只负责调度**：Loop 只做流程控制，不实现 Context 拼接、反馈分析、停机判断等逻辑
3. **Finish 虚拟工具**：`ToolCall(name="finish")` 不进入 Dispatcher，Loop 直接识别并交由 `AutonomousStopDecision.on_finish()`

**构造注入：**
```python
class HarnessLoop:
    def __init__(
        self,
        provider: LLMProvider,
        dispatcher: ToolDispatcher,
        guardrail: Guardrail,
        context_manager: ContextManager,
        stop_decision: AutonomousStopDecision,
        memory_store: MemoryStore | None = None,
        config: Config | None = None,
    ) -> None: ...
```

**主循环流程：**
```python
def run(self, task: str, tool_defs: list[ToolDef]) -> RunResult:
    ctx = self.context_manager.build(task, tool_defs)
    while True:
        response = self.provider.generate(ctx.messages, tool_defs, config)
        for tool_call in response.tool_calls:
            if tool_call.name == "finish":
                decision = self.stop_decision.on_finish(tool_call)
                return RunResult(success=decision.success, iterations=ctx.iteration, ...)
            guard = self.guardrail.check_tool_call(tool_call)
            if not guard.allowed:
                return RunResult(success=False, error=guard.reason, ...)
            result = self.dispatcher.dispatch(tool_call)
            ctx = self.context_manager.append_tool_result(ctx, tool_call, result)
            if result.exit_code is not None:  # 测试命令
                feedback = self.feedback_engine.analyze(result, ctx)  # T18 接入
                if feedback:
                    ctx = self.context_manager.append_feedback(ctx, feedback)
        decision = self.stop_decision.should_stop(ctx)
        if decision.should_stop:
            return RunResult(success=decision.success, iterations=ctx.iteration, ...)
```

**注意：** `feedback_engine` 部分在 T11 中可以先**跳过**（注释标记待 T18 接入），T11 只测试不包含 Feedback 的基础流程。

**注册工具到 Dispatcher：** 在 `src/harness/__init__.py` 中注册 ReadFile、WriteFile、RunShell 到 Dispatcher，供 Loop 使用。

### FailureAnalyzer 类

**职责：** 解析 pytest 输出，分类失败类型。

```python
class FailureAnalyzer:
    """Classifies test failures by parsing pytest output."""

    def analyze(self, collected: CollectedOutput) -> AnalysisResult:
        """Determine the failure type from collected output.

        Uses regex/pattern matching on pytest stdout/stderr to classify:
        - SYNTAX_ERROR: Python syntax errors
        - IMPORT_ERROR: Import errors
        - ASSERTION_ERROR: Assertion failures (extract expected/actual)
        - TIMEOUT: Execution timeout
        - RUNTIME_ERROR: Other runtime exceptions
        - TEST_FAILURE: General test failure (unclassified)
        - UNKNOWN: Cannot classify
        """
        ...
```

**分类规则（正则匹配）：**
- `SYNTAX_ERROR`：匹配 `SyntaxError:` 或 `SyntaxError: invalid syntax` 等
- `IMPORT_ERROR`：匹配 `ModuleNotFoundError:` 或 `ImportError:`
- `ASSERTION_ERROR`：匹配 `AssertionError:`、`assert ... == ... failed`，提取 `assert` 表达式和预期/实际值
- `TIMEOUT`：匹配 `Timeout:` 或 `timeout` 等关键词
- `RUNTIME_ERROR`：匹配 `Error:`、`Exception:`、`Traceback` 等（排除已分类的）
- `TEST_FAILURE`：匹配 `FAILED` 或 `failed` 测试名称
- `UNKNOWN`：兜底
